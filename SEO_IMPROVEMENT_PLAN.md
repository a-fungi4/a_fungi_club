import { ImageResponse } from 'next/og';

export const runtime = 'edge';

## Phase 4 — Image Alt Text Audit

**Context:** There are approximately 67 `<Image>` usages across the codebase. Currently only ~7 have alt text. Every `<Image>` must have an `alt` attribute.

### Rules
- If an image conveys information (thumbnail, photo, icon with meaning): write a descriptive alt string.
- If an image is purely decorative (background texture, abstract shape): use `alt=""` (empty string, not omitted).
- Never omit the `alt` attribute entirely.

### How to find all images missing alt text

Run this in the project root:
```bash
grep -rn "<Image" src/ | grep -v 'alt='
```

Each result is a line that needs an `alt` attribute added.

### Known images with correct alt text (already done — do not change these)

These are in `src/app/portfolio/PortfolioInteractive.tsx`:
- `1-UIUserResearch.webp` — has alt
- `2-UIWireFramingCram.webp` — has alt
- `3-UIPrototypingAndAnimation.webp` — has alt
- `1-BrandingArea59.webp` — has alt
- `2-BrandingDevelopingGripNDrip.webp` — has alt
- `3-BrandingSqueezyDawgs.webp` — has alt
- `1-MarketingVochos.webp` — has alt
- `2-MarketingTheCreationOfEarl.webp` — has alt
- `3-MarketingPresentingMyArt.webp` — has alt

### Suggested alt text for known images in other files

| File | Image src | Suggested alt |
|------|-----------|---------------|
| `src/app/layout.tsx` (noscript pixel) | Facebook pixel | `""` (decorative, tracking pixel) |
| `src/app/about/page.tsx` headshot | `/headshot-bw.webp` | `"Black and white headshot of Khaled Momani, designer and developer"` |
| Any project carousel images | `/design-to-code-carousel-photos/*` | `"Design to Code project screenshot — [describe what's shown]"` |
| Any project carousel images | `/heirloom-carousel-photos/*` | `"Heirloom app screenshot — [describe what's shown]"` |
| Icons used as images | Any icon | `""` if decorative, or describe if meaningful |

### How to add alt text

Find each `<Image` tag missing `alt`. Add `alt="your description here"` as an attribute. Example:

Before:
```tsx
<Image src="/headshot-bw.webp" width={300} height={300} />
```

After:
```tsx
<Image src="/headshot-bw.webp" width={300} height={300} alt="Black and white headshot of Khaled Momani, designer and developer" />
```

## Phase 5 — Core Web Vitals

### Task 5.1 — Add `priority` to above-the-fold images

**Rule:** Any image visible without scrolling when the page loads should have `priority` added.

Find the hero/banner images in:
- `src/components/HeroSection.tsx` — the main hero image
- `src/app/about/page.tsx` — the headshot image
- Any banner section image that loads immediately

Add `priority` attribute:
```tsx
<Image src="/headshot-bw.webp" ... priority />
```

Do NOT add `priority` to images inside carousels, grids, or below-the-fold sections.

### Task 5.2 — Ensure all `<Image>` components have dimensions

Every `<Image>` must have either:
- Explicit `width` and `height` props, OR
- `fill` prop with a sized parent container

Missing dimensions cause Cumulative Layout Shift (CLS), which hurts Core Web Vitals scores. Find any `<Image>` missing both `width`/`height` and `fill` using:

```bash
grep -rn "<Image" src/ | grep -v 'width=' | grep -v 'fill'
```

Fix each result by adding appropriate `width` and `height` values.
